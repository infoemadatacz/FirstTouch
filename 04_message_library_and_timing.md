# First Touch
## Message Library and Timing (Unified EMA + First Touch Positioning)

Last update: 2026-02-24

## 1. Timing Map

- Day 0: Connection request.
- Day 1-2: Thank-you + relevance note.
- Day 4-5: Insight/asset value message.
- Day 7-9: Diagnostic question.
- Day 10-14: Working-call invitation.
- Day 18-21: Respectful close-the-loop message.

## 2. Identity Rule for DMs

Default:
- Start all outreach as EMA AI CEO/operator.
- Introduce First Touch later as framework/mechanism.

Do not open with:
- "I am building a separate First Touch venture."
- "I run another project called First Touch."

## 3. Connection Request Scripts

Keep under 280 characters.

### CEO/COO

"Hi {FirstName}, I'm building EMA AI for financial services.  
I share practical work on executive throughput and workflow orchestration (First Touch).  
Your work on {initiative} stood out. Open to connect?"

### RevOps/CS/Ops

"Hi {FirstName}, I publish practical operators' playbooks on intake-to-execution workflows in regulated teams.  
Would value connecting."

### IT/Security

"Hi {FirstName}, I work on governance-first autonomous workflow design inside EMA AI initiatives.  
Would like to connect and exchange practical patterns."

## 4. First Message After Acceptance (No Pitch)

"Thanks for connecting, {FirstName}.  
I'm currently mapping where leadership teams lose throughput between incoming requests and execution.  
Curious how this shows up in your world at {Company}."

## 5. Value Message (Day 4-5)

### Option A

"One pattern we keep seeing: teams automate outputs before redesigning triage.  
That creates speed but weak control.  
Happy to share the one-page triage checklist if useful."

### Option B

"I use a simple workflow map:  
`Detect -> Classify -> Route -> Resolve -> Audit`  
It helps define where autonomy is safe and where approval is required.  
Want me to send it?"

## 6. Diagnostic Question (Day 7-9)

Ask one question only.

### CEO/COO

"Where do senior leaders still intervene too often: triage, approvals, or exception handling?"

### RevOps/CS/Ops

"Which handoff currently creates the most rework?"

### IT/Security

"What is the biggest blocker to safe autonomous execution in your current workflows?"

## 7. First Touch Introduction (Only After Intent Signal)

Use only after they replied with relevant pain:

"Inside EMA AI work, we use an orchestration framework called First Touch for this exact bottleneck.  
Happy to share how we scope it for one workflow if useful."

This keeps separation clear:
- EMA = primary business identity.
- First Touch = named operating framework.

## 8. Call Invitation (Day 10-14)

Only after at least one positive signal.

"If useful, we can run a focused 25-minute working session.  
We'll map one high-friction workflow, define autonomy thresholds, and identify a constrained pilot boundary.  
No deck, just practical design. Open to it next week?"

Time option:

"I can do Tue 11:00 CET or Thu 15:30 CET for 25 minutes."

## 9. Branching

### Positive reply

"Perfect.  
To make it practical, which workflow should we map first?"

### Vague reply

"Makes sense.  
Would a short 5-question pre-call checklist help us keep it concrete?"

### No reply after call invite

Wait 5-7 days:

"I'll close the loop here so I don't spam your inbox.  
If this becomes a priority this quarter, happy to run that 25-minute workflow map anytime."

## 10. Long-Cycle Relationship Messages (Every 2-3 Weeks)

1. "Saw your update on {initiative}. Curious what changed operationally after launch?"
2. "We're seeing governance design move ahead of tooling decisions in regulated teams. Are you seeing the same?"
3. "One practical lesson from recent workflow maps: fewer automation points often drive faster adoption."

## 11. Quality Rules

- One CTA per message.
- Keep messages short (max 5 lines).
- Reference one concrete detail from their context.
- No attachments unless requested.
- Keep tone peer-to-peer, not vendor pitch.
- Keep EMA identity explicit in early thread.
- Use First Touch label only when useful for clarity.

## 12. Proposal Escalation Rule

Move from call to proposal only if:
- Workflow pain is explicit.
- Operational owner is known.
- Pilot boundary and success metric are agreed.

Proposal structure:
- 2-week diagnostic.
- 6-8 week constrained pilot.
- ROI + governance scorecard.
