# First Touch
## Sales Call Playbook (25-Min Working Session)

Last update: 2026-02-24

## 1. Call Objective

By minute 25, decide:
- clear pilot candidate workflow?
- owner and priority confirmed?
- diagnostic sprint proposal yes/no?

## 2. Call Agenda

- 0:00-2:00 Context and objective
- 2:00-8:00 Bottleneck discovery
- 8:00-14:00 Current workflow mapping
- 14:00-19:00 Threshold and governance boundaries
- 19:00-23:00 Pilot scope recommendation
- 23:00-25:00 Next-step close

## 3. Opening Script

"Thanks for joining. This is a working session, not a deck review. We will map one workflow bottleneck and decide if there is real pilot value."

## 4. Discovery Questions

- Which workflow causes most leadership intervention?
- Where do handoffs break most often?
- What is currently measured and what is not?
- Which decisions can never be autonomous?
- Who owns this workflow operationally?

## 5. Mapping Questions

- What starts the workflow?
- How is priority currently decided?
- Where does rerouting happen?
- What are common exception types?
- What is considered a successful resolution?

## 6. Governance Questions

- Which data is sensitive in this flow?
- Which step requires mandatory human sign-off?
- What would compliance need for auditability?

## 7. Qualification Check at Minute 20

Confirm all three:
- active quarter priority
- named owner
- constrained pilot willingness

If yes, move to proposal step.
If no, close with no-pressure next step.

## 8. Proposal Close Script

"Based on this map, the next step is a 2-week diagnostic sprint. We will baseline current performance, define thresholds, and produce pilot scope with KPI scorecard. If useful, I can send the proposal today."

## 9. If Prospect Pushes for Price First

"We can share low/mid/high anchors now. Final pricing depends on workflow complexity and governance scope."

## 10. No-Pressure Close Script

"If this is not active this quarter, no issue. I can share the map template and we can revisit when the timing is right."

## 11. Post-Call Follow-Up (within 2h)

Send:
- 5-bullet summary
- identified bottleneck
- proposed next step
- timeline
- explicit CTA (approve proposal / pause)
