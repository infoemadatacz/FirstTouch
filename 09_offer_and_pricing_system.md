# First Touch
## Offer and Pricing System

Last update: 2026-02-24

## 1. Offer Architecture

### Offer 0: Workflow Bottleneck Map Call
- Duration: 25 minutes
- Goal: Identify one constrained pilot candidate workflow
- Price: free
- Output: quick map + go/no-go recommendation

### Offer 1: Diagnostic Sprint
- Duration: 2 weeks
- Goal: establish baseline, pilot scope, governance thresholds
- Output:
  - workflow map
  - baseline KPI set
  - autonomy threshold matrix
  - pilot implementation plan

### Offer 2: Constrained Pilot
- Duration: 6-8 weeks
- Goal: run one workflow lane with measurable outcomes
- Output:
  - orchestration runbook
  - weekly KPI reporting
  - exception analysis
  - scale recommendation

### Offer 3: Scale Program
- Duration: quarterly
- Goal: add lanes, integrations, governance depth
- Output:
  - expanded orchestration map
  - policy package
  - operating review cadence

## 2. Pricing Bands

Use one of these based on market maturity and account size.

### Diagnostic Sprint
- Low anchor: EUR 7,500
- Mid anchor: EUR 12,500
- Premium anchor: EUR 20,000

### Constrained Pilot
- Low anchor: EUR 25,000
- Mid anchor: EUR 45,000
- Premium anchor: EUR 80,000+

### Scale Program (monthly)
- Low anchor: EUR 8,000
- Mid anchor: EUR 15,000
- Premium anchor: EUR 30,000+

## 3. Price Selection Logic

Use higher anchor when:
- high compliance complexity
- multi-system integrations
- multiple stakeholder groups
- urgent executive priority

Use lower anchor when:
- single team, single lane, low integration depth

## 4. Commercial Terms

- 50% upfront, 50% on midpoint milestone for diagnostics and pilots
- Change requests outside agreed scope billed separately
- Success metrics confirmed before kickoff
- Explicit client owner required

## 5. Packaging Rules

Never sell:
- broad "AI transformation" without constrained scope

Always sell:
- one lane
- one owner
- one baseline
- one measurable outcome path

## 6. Outcome Framing

Focus on:
- cycle time reduction
- executive intervention reduction
- rework reduction
- exception handling quality

Avoid:
- vanity claims about "AI adoption"

## 7. Proposal Components

Every proposal must include:
1. business context and workflow pain
2. baseline assumptions
3. scope boundary
4. governance boundary
5. KPI scorecard
6. timeline and milestones
7. commercial terms

## 8. Red Flags (Do Not Engage)

- No owner and no executive sponsor
- Wants broad automation without governance
- "Just show us demo" with no active pain
- Wants guaranteed outcome with no process change

## 9. Expansion Strategy

After successful pilot:
- propose adjacent lane with shared bottleneck type
- expand where tooling overlap is high
- keep governance model consistent

## 10. One-Line Offer Statement

"We map one high-friction workflow, run a constrained orchestration pilot, and prove throughput impact with governance-safe execution."
