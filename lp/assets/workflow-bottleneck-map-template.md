# Workflow Bottleneck Map Template

Use this before any AI workflow implementation.

## 1) Workflow Name
- Example: Executive escalation triage, sales inbound triage, compliance request routing

## 2) Current Intake Paths
- Where requests come in today (email, chat, CRM, tickets, docs)
- Which channels create the most noise

## 3) Ownership Map
- Who currently decides priority
- Who currently routes tasks
- Who approves final actions

## 4) Failure Points
- Where rework happens
- Where requests are dropped or delayed
- Where leadership gets pulled in too often

## 5) Decision Boundaries
- What can be auto-executed safely
- What must be drafted for human approval
- What must always escalate to human owner

## 6) Policy and Risk Notes
- Sensitive data fields involved
- Compliance-critical checkpoints
- Audit requirements

## 7) Baseline Metrics (Week 0)
- Intake-to-resolution cycle time
- Rework rate
- Escalation rate
- Executive intervention count

## 8) Pilot Scope
- One specific workflow lane only
- Start date and end date
- Pilot owner

## 9) Week 1-4 KPI Tracking
- Weekly trend on the four baseline metrics
- Exceptions and causes
- Next adjustment decisions

## 10) Scale Decision
- Scale now / iterate / stop
- Why
- What must be fixed before scale
